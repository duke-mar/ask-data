# AskData - AI 数据问数平台

基于 **AI + SQL + 自然语言** 的数据可视化分析平台。用户通过自然语言提问，系统自动完成意图识别、SQL 生成、安全检查、查询执行、图表渲染和数据分析结论输出。

## 适用于学习和小团队使用
  1. 该项目仅仅作为一个web工具提供服务，即用户在使用时，数据库信息、大模型API KEY信息都存储在本地localstorage，对于使用者而言仅仅是用自己的资源使用工具
  2. 若向做成SaaS平台提供服务，则需添加账号权限系统、数据隔离机制等考虑，感兴趣的开发者自行摸索

## 功能特性

- **数据库管理**：连接 MySQL 数据库，选择关联数据表，读取表结构 schema
- **大模型配置**：支持任意 OpenAI 兼容格式的 API（可自定义 API 地址、Key、模型）
- **AI 问数全流程**：
  1. **意图识别** — 双层校验判断是否为有效的数据查询指令
  2. **SQL 生成** — AI 根据 schema 生成 MySQL 查询语句（SSE 流式）
  3. **安全检查** — 拦截危险操作、未授权表查询
  4. **自动执行** — 执行查询，错误时自动修正 SQL（最多重试 3 次）
  5. **图表渲染** — AI 生成 ECharts 配置，前端渲染交互式图表
  6. **数据分析** — AI 给出数据洞察和业务建议（SSE 流式）
  7. **报告导出** — 支持导出 Markdown 报告
- **流式输出**：SQL 生成和分析结论均支持流式输出，实时可见
- **处理计时器**：任务处理中动态显示耗时，避免用户等待焦虑
- **时间轴侧边栏**：每个步骤的执行状态和时间轴记录，成功/失败一目了然
- **表结构备注**：为数据表添加描述，帮助 AI 更好理解数据库
- **会话管理**：多会话切换，聊天记录持久化（localStorage）

## 使用指南

### 第一步：配置大模型 API

1. 点击左侧面板底部的「配置大模型 API」
2. 填写 API 地址、API Key 和模型名称
3. 保存后自动进行健康检查，顶部显示绿色 ✓ 表示可用

**支持的模型**（OpenAI 兼容格式）：

| 服务商 | API 地址示例 | 模型名称 |
|---|---|---|
| OpenAI | `https://api.openai.com/v1/chat/completions` | `gpt-4o` |
| DeepSeek | `https://api.deepseek.com/chat/completions` | `deepseek-chat` |
| Azure OpenAI | 你的部署地址 | 你的部署名 |
| Ollama（本地） | `http://localhost:11434/v1/chat/completions` | `qwen2.5` |

### 第二步：添加数据库连接

1. 点击左侧面板「数据库连接」右侧的 `+` 按钮
2. 填写 MySQL 连接信息（主机、端口、用户名、密码、数据库名）
3. 点击「测试连接」验证
4. 点击「保存连接」

### 第三步：选择数据表

1. 在左侧面板点击已添加的数据库连接，选中它
2. 在「选择数据表」区域勾选需要关联的表（**最多 10 个**）
3. （可选）展开表结构，添加备注帮助 AI 理解字段含义

### 第四步：开始问数

在右下角输入框输入自然语言问题，按 Enter 或点击发送按钮。

## 如何输入有效指令

### ✅ 有效的问题示例

| 类型 | 示例 |
|---|---|
| 明细查询 | "查询用户表中前 10 条记录" |
| 条件查询 | "查找年龄大于 30 岁的用户" |
| 统计分析 | "统计各品类的销售额占比" |
| 趋势分析 | "按月份统计订单量趋势" |
| 排名分析 | "销售额排名前 5 的商品" |
| 关联查询 | "查询订单对应的用户信息" |
| 聚合计算 | "计算每个用户的平均订单金额" |
| 分组统计 | "按地区统计用户数量" |

### ❌ 会被系统拦截的指令

以下输入会被意图识别或安全检查拦截，**不会生成 SQL、不会执行查询**：

| 输入 | 系统响应 |
|---|---|
| "你好" / "在吗" / "hello" | 问候语会被识别，提示输入具体的数据查询问题 |
| "帮我写首诗" | 非数据查询指令，提示输入有效的数据查询问题 |
| "删除所有用户" | **危险操作** — 意图识别阶段直接拦截，不会调用大模型生成 SQL |
| "把订单表清空" | **危险操作** — 意图识别阶段直接拦截 |
| "插入一条新记录" | **危险操作** — 意图识别阶段直接拦截 |
| "修改用户年龄为 18" | **危险操作** — 意图识别阶段直接拦截 |
| "删表" / "drop table" | **危险操作** — 意图识别阶段直接拦截 |
| "查询 information_schema" | 系统表禁止查询，SQL 安全检查拦截 |

### 🛡️ 安全机制说明

系统采用**双层安全校验**，危险操作在**最早阶段**就被拦截：

**第一层：意图识别（本地快速检查 + AI 二次确认）**

- **本地检查**（不调用大模型，毫秒级响应）：
  - 检测问候语、闲聊、指令不完整
  - **检测数据修改意图**：含有 `删除/清空/drop/插入/新增/修改/更新/替换/创建表` 等关键词，且不含 `查询/查看/统计/分析` 等查询语境词时，直接拦截
  - 检测 prompt injection 攻击模式
- **AI 检查**（调用大模型兜底）：
  - 让大模型判断输入是否为有效的数据查询/统计指令
  - 同时判断是否为数据修改/危险操作意图
  - 无效时给出具体原因和正确示例

> 💡 **为什么这样设计？** 如果用户输入"删除所有用户"，系统在本地检查阶段就能识别出危险意图，**不会调用大模型生成 SQL**，既节省 Token 又让用户安心。

**第二层：SQL 安全检查（生成 SQL 后的兜底校验）**

即使通过了意图检查，生成 SQL 后仍会进行：
- 拦截 `INSERT/DELETE/UPDATE/DROP/TRUNCATE/ALTER/CREATE/GRANT/REVOKE` 等危险操作
- 禁止查询 `information_schema/mysql/performance_schema/sys` 等系统表
- SQL 中引用的表必须在左侧已选择的表列表中

### 💡 输入技巧

1. **明确表达查询意图**：使用"查询"、"统计"、"分析"、"查看"等词开头，系统更容易识别
2. **避免危险动词**：如需查询与删除/修改相关的数据，请用查询语境表达，例如：
   - ✅ "**查询**已**删除**的订单"（含"查询"，系统判断为正常查询）
   - ❌ "**删除**所有订单"（无查询语境，被拦截）
3. **尽量具体**："统计最近 7 天各品类的销售额" 比 "看看销售情况" 效果更好
4. **表名和字段名**：如果 AI 对表结构理解有误，可在左侧表结构中添加备注

## 核心交互流程

```
用户提问
  ↓
[Step 0] 意图识别 — 判断是否为有效的数据查询/统计指令
         ❌ 无效（问候语/闲聊/指令不完整/危险操作）→ 友好提示，终止流程
         ✅ 有效 → 继续
  ↓
[Step 1] 组装 Prompt = 用户问题 + 选中数据库 schema（含用户备注）
  ↓
[Step 2] 调用 /api/llm/generate-sql → SSE 流式获取 SQL
  ↓
[Step 3] 安全检查 — 拦截危险操作、未授权表查询
         ❌ 不通过 → 报错终止
         ✅ 通过 → 继续
  ↓
[Step 4] 执行查询。若报错 → 报错信息 + 原输入再次请求（最多 3 次）
  ↓
[Step 5] 判断是否需要图表分析
         纯查询（如"前 5 条数据"）→ 只返回表格，跳过图表
         统计/分析类 → 继续生成图表
  ↓
[Step 6] 调用 /api/llm/generate-chart → 获取 ECharts option
  ↓
[Step 7] 调用 /api/llm/analyze → SSE 流式获取分析结论
  ↓
显示「导出 Markdown」按钮
```

## 会话管理

- **自动创建**：发送第一条消息时自动创建会话，标题 = 第一条指令内容
- **手动创建**：点击会话管理右上角的 `+` 创建新会话
- **切换会话**：点击会话列表切换，每个会话有独立的聊天记录和时间轴
- **删除会话**：悬停后点击 🗑️ 删除
- **重命名**：双击会话标题编辑

**刷新页面后**：
- 所有会话保留（存储在 localStorage）
- 自动恢复上次活跃的会话
- 时间轴历史也一并恢复

## 项目结构

```
app/
├── page.tsx                    # 主页面（四栏布局）
├── layout.tsx                  # 根布局
├── globals.css                 # 全局样式（石色系主题）
├── types/index.ts              # 全局类型定义
├── lib/
│   ├── db.ts                   # MySQL 连接封装
│   ├── llm.ts                  # 大模型 API 封装（SSE 流式 + 非流式）
│   ├── prompts.ts              # Prompt 模板（SQL/图表/分析/意图识别）
│   ├── schema.ts               # Schema 格式化 + 查询结果格式化
│   ├── storage.ts              # localStorage 封装
│   └── utils.ts                # 工具函数 + SQL 安全检查 + 意图校验
├── hooks/
│   ├── useStream.ts            # SSE 流式读取 Hook
│   ├── useDbConnection.ts      # 数据库连接管理
│   ├── useChat.ts              # 对话状态 + AI 全流程
│   └── useTimeline.ts          # 时间轴步骤管理
├── api/
│   ├── db/connect              # POST - 测试数据库连接
│   ├── db/tables               # GET - 获取表列表
│   ├── db/schemas              # GET - 获取表结构
│   ├── db/query                # POST - 执行 SQL 查询（后端安全检查）
│   ├── llm/chat                # POST SSE - 通用流式对话
│   ├── llm/generate-sql        # POST SSE - 生成 SQL
│   ├── llm/generate-chart      # POST - 生成 ECharts option
│   ├── llm/analyze             # POST SSE - 生成分析结论
│   └── export/markdown         # POST - 导出 Markdown 报告
└── components/
    ├── sidebar/                # 侧边栏组件（数据库配置、表选择、时间轴、会话）
    ├── chat/                   # 对话组件（消息列表、输入框、计时器）
    ├── chart/                  # 图表组件
    └── export/                 # 导出组件
components/ui/                  # Shadcn/ui 基础组件
```
